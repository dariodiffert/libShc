Installation

Content:

The libShc.zip contains

  • build: After running make, the library is build into this folder.
  • examples: After running make, the examples are build into this folder. Note that all examples have to be started from within this folder due to the usage of local paths to the material folder.
  • include: The header files are located in this folder.
  • kissfft: The external library "kissFFT" to compute fast Fourier transforms is located in this folder.
  • manual: Location of this manual.
  • src: The source files (*.cpp) of the libShc and its extension libShx (including support for loading/saving images from the drive).
  • src_examples: The source files (*.cpp) of the examples.

Dependencies:

The libShc heavily relies on "Eigen" for matrix and vector computations (see Literature). Moreover, the optional class Shx depends on "OpenCV" for reading/writing images from/to the drive (see Literature). We chose OpenCV since it is widely spread in scientific community, however it can be simply replaced by an arbitrary image library by only changing a few lines (they are marked) of code in Shx.h and Shx_wrapper.cpp. Furthermore the standard Unix library "The Realtime Extension" is used. To use this library under windows make sure to replace the calls for time measurements.

Compiling:

To compile enter the root folder ./libShc/ and run make. The resulting libraries libShc.a and libShx.a are created in the "build" folder, the headers can be found in the "include" folder.

Quick Start:

Enter the directory libShc/ and enter the following to the console.

1 echo "Install dependency: Eigen (required)"
2 sudo apt-get install libeigen3-dev
3 echo "Install dependency: OpenCV (optional, required for direct image access via the class Shx)"
4 sudo apt-get install libopencv-dev
5 echo "Compile libShc"
6 make
7 echo "Done. Check out the examples/ directory!"

Examples:

After compiling, the examples can be found in ./libShc/examples/. Since local paths are used, make sure to run the examples from their local folder.
