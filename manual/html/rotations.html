Rotations

The spherical harmonics are mathematically strongly related to rotations and behave well under rotational operations. Make sure to use the correct Coordinate System for rotations.

In this toolbox several methods are combined to reduce the computation time for both the rotation matrices and performning rotations:

  • Sparsity: Several sparsity relations (see below) are used.
  • Hemispherical Continuation: Some Hemispherical Continuations furthermore increase the sparsity of the rotation matrices.
  • Coarse-to-Fine: Multiple rotations can be performed one after another (with increasing accuracy) to reduce the total number of rotation matrices needed.

However, the calculation of the rotation matrix itself can be — depending on the number of bands used — computational expensive. Hemispherical Continuations:

Additionally to the sparsity relations mentioned above, the sparsity relations induced by the hemispherical continuation speed up all rotations. See Hemispherical Continuations for more information.

Coarse-to-Fine:

By calling shc::Shc::init_rotations_sphere, shc::Shc::init_rotations_cone, and shc::Shc::init_rotations_custom, rotation matrices can be precalculated.

Each one adds a set of rotation matrices to the shc::Shc instance. By calling any function related to rotations, all sets of rotations are used in the order they were initialized to perform the rotation.

For example you can first add a set of rotations covering angles up to in steps and afterwards add another set of finer rotations covering angles up to in steps. Multiple sets of rotations always increase the computation time to compute rotations via shc::Shc::rotate, while they decrease the computation times for shc::Shc::compass.
