Translations (Warping)

Translations can be simulated in the basis of spherical harmonics by changing the center of the spherical harmonics. This is done by approximating the translation in spatial space and transforming the result into the basis of spherical harmonics, where it can be written as a matrix multiplication. This process is computational expensive and is therefore precalculated only for Z-axis translations during the initialization phase (see shc::Shc::init). Arbitrary translations can then be calculated by concatenating rotations and Z-axis translations using shc::Shc::warp. We refer to translations in arbitrary directions as ''warp'' or ''warping''.

Note that this approach does not work with Hemispherical Continuations. In this case, the required translation matrices have to be computed individually via shc::Shc::create_matrix_warp.

Interpretations:

Translations can be interpreted differently. In this example we interpret the panoramic image as a density distribution (of points, particles, or similar). This can be noticed by observing the following effects:

  • The area around the point of expansion gets darker for larger translations: A smaller number of points can be observed in this direction.
  • The area around the point of contraction gets lighter for larger translations: A larger number of points can be observed in this direction.
  • Moving outside the sphere ( ) superimposes the distribution of the complete sphere: All particles are behind the observer.

We call this kind of translation DENSITY. In contrast, a purely VISUAL interpretation is basically similar but does not show the special effects mentioned above. Both interpretations can be used by setting the corresponding parameter during the initialization via shc::Shc::init_translations.

Pointclouds:

The DENSITY interpretation can be used for example to align Pointclouds (e.g. 3D model data). The libShc provides the functionality to load and manipulate Pointclouds.

Slices:

In the standard case a single sphere (in the following called slice) is used to represent a function defined on the unit sphere. However, by projecting three-dimensional data on the sphere valuable data can be lost. Instead of projecting onto a single slice with radius we can project onto multiple slices with differing radii.

The number of slices can be set in the initialization phase via shc::Shc::init_slices. Note that the input function for translations as well as the result of translations will be a vector with slices stored in a vector of shc::Shpm instances.

Problem Cases:

Translations (especially with multiple slices) are an experimental feature, therefore note the following points:

  • Translations are precalculated for the values set by shc::Shc::init_translations during the initialization phase. Since they are approximated in spatial space, the quality is determind by the number of sampling points set by shc::Shc::init_surface
  • Translations contract the function around the point of contractions, which increases the number of necessary bands to express the translated function.
  • Translations using multiple slices seems reasonable for the DENSITY interpretation only.
  • By using slices, each translation is not only expressed by one but transformation matrices. The calculation time increases by the factor .
